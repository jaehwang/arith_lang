## Codebase Patterns
- Function AST nodes live in include/function_ast.h (separate from ast.h); codegen stubs in src/function_codegen.cpp
- FunctionLiteralAST inherits ExprAST; ReturnStmtAST inherits ASTNode (not ExprAST, it's a statement)
- Token enum values for keywords use negative integers (e.g., TOK_FN = -14); single-char operators use ASCII values
- Keyword recognition is in `handleKeywordOrIdentifier()` in src/lexer.cpp; operator recognition in `handleOperator()`
- Multi-char operators (like `=>`, `==`, `>=`) handled in `handleOperator()` by checking next char after consuming first
- AST nodes are in `include/ast.h`; codegen in `src/codegen.cpp`; parser in `src/parser.cpp`/`include/parser.h`
- Integration test files go in `tests/k/` with `// EXPECTED:` comments; run via `./test_runner.sh` from project root
- Unit tests use Google Test; individual test files in `tests/test_*.cpp`; run via `cd build && ctest`
- Parser tracks `functionDepth` (private int) to distinguish return inside function body from top-level return
- Function calls use postfix parsing: parsePostfixExpr() wraps parsePrimary() in a loop checking for TOK_LPAREN; parseUnaryExpr() calls parsePostfixExpr() instead of parsePrimary()
- Type checker uses TypeInfo {ValueType type, int param_count} to propagate function arity from inferExprType; SymbolInfo has is_parameter flag to block immutable param reassignment
- Type checker handles FunctionLiteralAST in inferExprType: validates dup params, capture validity, then enters function scope, declares params, type-checks body, exits scope
- Function pointers stored as double in allocas via ptrtoint (ptrâ†’i64) + bitcast (i64â†’double); decoded by bitcast (doubleâ†’i64) + inttoptr (i64â†’ptr) before indirect calls
- FunctionLiteralAST::codegen() saves/restores builder insert block around inner function body generation; uses cg.enterScope()/exitScope() for param scope
- Indirect LLVM calls require explicit FunctionType* in CreateCall(funcType, fnPtr, args) with opaque pointers (LLVM 15+)
