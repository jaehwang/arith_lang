# ArithLang TODO 목록 (우선순위 순)

## 🎯 개발 우선순위

### 1. **함수 시스템 구현** 🔥 **최고 우선순위**
- **이유**: `plans/sample-function-implementation-plan.md`에 완전한 구현 계획이 이미 준비되어 있음
- **영향도**: 언어의 표현력을 크게 향상시키는 핵심 기능
- **준비도**: 상세한 구현 계획, 테스트 전략, 단계별 로드맵이 모두 완비됨
- **구현 범위**:
  - `fn(x, y) => x + y` 문법의 함수 리터럴
  - 렉시컬 클로저 (immutable/mutable 캡처)
  - 함수 호출 및 고차 함수 지원
  - 기존 변수 시스템과의 완전한 통합
- **예상 작업 기간**: 3-4주 (계획서 기준)
- **완료 시 효과**: 
  - 함수형 프로그래밍 지원
  - 코드 재사용성 증대
  - 고차 함수를 통한 표현력 향상

### 2. **향상된 오류 진단 시스템** 🚀 **높은 우선순위**  
- **이유**: 현재 기본적인 오류 위치 보고는 완료되었지만, 더 개선 가능한 영역
- **영향도**: 개발자 경험을 크게 향상시키고 언어 사용성 증대
- **구현 범위**:
  - 다중 오류 보고 (첫 번째 오류에서 멈추지 않고 계속 파싱)
  - 오류 복구 메커니즘 강화
  - IDE 통합을 위한 LSP (Language Server Protocol) 지원
  - 컬러 출력 및 더 상세한 힌트/제안 시스템
- **예상 작업 기간**: 2-3주
- **완료 시 효과**:
  - 디버깅 효율성 증대
  - IDE 통합 가능성
  - 초보자 친화적인 오류 메시지

### 3. **모듈 시스템** 🔮 **중간 우선순위**
- **이유**: `specs/modules.md`에 명세가 있으며, 대규모 프로그램 작성을 위해 필요
- **영향도**: 코드 재사용성과 프로젝트 확장성 향상
- **구현 범위**:
  - `import`/`export` 문법 
  - 모듈 별 네임스페이스 관리
  - 의존성 해결 및 순환 참조 검사
  - 컴파일 단위 분리 및 링킹
- **예상 작업 기간**: 4-5주
- **완료 시 효과**:
  - 대규모 프로젝트 지원
  - 라이브러리 생태계 구축 가능
  - 코드 조직화 개선

## 📋 선정 기준

1. **준비도**: 이미 계획이나 명세가 있는 항목 우선
2. **영향도**: 언어의 표현력과 사용성에 미치는 영향
3. **복잡도**: 기존 아키텍처와의 통합 난이도
4. **연관성**: 기존 구현된 기능들과의 시너지

## 🏗️ 구현 전략

### 단계별 접근 방식
1. **1단계**: 함수 시스템의 기본 구조 구현 (1-2주)
2. **2단계**: 클로저와 고급 기능 완성 (1-2주)  
3. **3단계**: 오류 진단 시스템 개선 (2-3주)
4. **4단계**: 모듈 시스템 설계 및 구현 (4-5주)

### 현재 프로젝트 상태
- ✅ **완전한 기반 시스템**: 모든 핵심 컴파일러 구성 요소 완료
- ✅ **100% 테스트 통과**: 31/31 통합 테스트, 91/91 네거티브 테스트
- ✅ **변수 시스템**: immutable-by-default with `mut` keyword
- ✅ **타입 체킹**: 완전한 타입 안전성 시스템
- ✅ **오류 위치 추적**: 파일:줄:열 형식의 정확한 오류 보고

### 권장 시작점
**함수 시스템 구현**부터 시작하는 것을 강력히 권장합니다:
- 완전한 구현 계획이 `plans/sample-function-implementation-plan.md`에 준비됨
- 기존 아키텍처와의 통합 방법이 명확히 정의됨
- 단계별 구현 가이드와 테스트 전략 포함
- 즉시 개발 착수 가능
