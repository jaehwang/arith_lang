## 🎯 현재 개발 우선순위

### 1. **향상된 오류 진단 시스템** 🔥 **최고 우선순위**  
- **이유**: 현재 기본적인 오류 위치 보고는 완료되었지만, 더 개선 가능한 영역
- **영향도**: 개발자 경험을 크게 향상시키고 언어 사용성 증대
- **구현 범위**:
  - 다중 오류 보고 (첫 번째 오류에서 멈추지 않고 계속 파싱)
  - 오류 복구 메커니즘 강화
  - IDE 통합을 위한 LSP (Language Server Protocol) 지원
  - 컬러 출력 및 더 상세한 힌트/제안 시스템
- **예상 작업 기간**: 2-3주
- **완료 시 효과**:
  - 디버깅 효율성 증대
  - IDE 통합 가능성
  - 초보자 친화적인 오류 메시지

### 2. **모듈 시스템** 🚀 **높은 우선순위**
- **이유**: `specs/modules.md`에 명세가 있으며, 대규모 프로그램 작성을 위해 필요
- **영향도**: 코드 재사용성과 프로젝트 확장성 향상
- **구현 범위**:
  - `import`/`export` 문법 
  - 모듈 별 네임스페이스 관리
  - 의존성 해결 및 순환 참조 검사
  - 컴파일 단위 분리 및 링킹
- **예상 작업 기간**: 4-5주
- **완료 시 효과**:
  - 대규모 프로젝트 지원
  - 라이브러리 생태계 구축 가능
  - 코드 조직화 개선

### 3. **배열/리스트 데이터 타입** 🔮 **중간 우선순위**
- **이유**: 데이터 처리 능력 향상을 위한 컬렉션 지원
- **영향도**: 실용적인 프로그램 작성 능력 대폭 증대
- **구현 범위**:
  - 배열 리터럴 문법: `arr = [1, 2, 3];`
  - 인덱싱 연산: `arr[0]`, `arr[i]`
  - 배열 연산: `length()`, `push()`, `pop()`
  - 배열 순회를 위한 for-each 루프
- **예상 작업 기간**: 3-4주
- **완료 시 효과**:
  - 데이터 처리 및 알고리즘 구현 가능
  - 실용적인 프로그램 작성 능력 향상
